# platform-diagnosis - 平台总体经营体检

## 概述

**用途**：分析平台整体经营数据（不限定VV渠道），包含新增·LTV、活跃、营收、留存四个维度的交叉诊断。

**版本**：v2.6（完整分平台数据提取）
**更新时间**：2026-06-26

## v2.6更新内容

### 核心改进

1. **分平台新增数据提取**：点击"新增"分类后提取各平台新增用户数据
2. **分平台活跃数据提取**：点击"活跃"分类后提取各平台活跃用户数据
3. **完整四维数据**：现在所有四个维度都有分平台数据
4. **平台排序适配**：每个分类的表格排序不同，需要根据数据值匹配平台名

### 修复内容

1. **修复变化率解析**：正确处理箭头符号（▲▼），准确识别上升/下滑方向
2. **降低阈值敏感度**：变化率阈值从3%降至1%，更敏感地捕捉信号
3. **新增诊断规则**：
   - 规则14: 获量下滑但营收增长（中风险）
   - 规则15: 活跃增长获量下滑（中风险）
   - 规则16: 高集中度风险（高风险，HHI≥5000）
4. **HHI纳入诊断信号**：集中度风险可触发诊断规则
5. **修复浏览器提取流程**：明确从 snapshot 直接读取数据（比 evaluate 更可靠）

## Agent 执行流程

```
用户请求 → Agent 执行以下步骤：

1. 运行检查脚本
   node scripts/run-platform-diagnosis.mjs
   
   ├─ 返回码 0 → 数据新鲜，继续处理
   ├─ 返回码 10 → 数据过期，执行浏览器提取
   └─ 返回码 1 → 其他错误，报告问题

2. 如果返回码 10（数据过期），执行浏览器提取：

   【步骤 2.1】启动浏览器
   browser: action=start, profile=openclaw
   
   【步骤 2.2】导航到产品数据页面
   browser: action=navigate, url=https://allcmsweb-pro.vvyyds.com/#/dataCenter/productData
   
   【步骤 2.3】等待加载，获取页面快照
   browser: action=snapshot
   
   【步骤 2.4】确保"全平台"按钮选中
   
   【步骤 2.5】从快照读取KPI数据（营收、新增、活跃、留存）
   
   【步骤 2.6】从快照读取平台营收数据（默认页面）
   
   【步骤 2.7】点击"新增"分类提取分平台新增数据
   
   【步骤 2.8】点击"活跃"分类提取分平台活跃数据
   
   【步骤 2.9】匹配平台名称（注意排序差异）
   
   【步骤 2.10】构建完整数据并保存
   
   【步骤 2.11】关闭浏览器

3. 重新运行诊断
   node scripts/run-platform-diagnosis.mjs

4. 返回结果
   - 输出文件路径
   - 健康分和关键诊断
```

| ID | 规则名 | 风险 | 假设 |
|----|--------|------|------|

| 2 | 获量规模下滑 | 高 | 新增↓ 活跃↓ |
| 3 | 活跃度塌方 | 高 | 活跃↓ 营收↓ |
| 4 | 鲸鱼依赖加剧 | 极高 | 活跃↓ 营收↑ 基尼≥0.6 |
| 5 | 付费转化下滑 | 高 | 营收↓ 付费率↓ |
| 6 | 存量用户流失 | 高 | 留存↓ 老留存主导 |
| 7 | 新留存拖累 | 中 | 留存↓ 新留存主导 |
| 8 | ARPPU提升对冲 | 中 | 付费用户↓ 营收→ |
| 9 | 规模扩张正常 | 低 | 四维全↑ |
| 10 | 营收下滑但用户稳 | 高 | 营收↓ 活跃→ |
| 11 | 全盘性下滑 | 极高 | 四维全↓ |
| 12 | 全维增长 | 低 | 四维全↑ |
| 13 | 稳中向好 | 低 | 营收↑ 其他稳定 |
| 14 | 获量下滑但营收增长 | 中 | 新增↓ 营收↑ |
| 15 | 活跃增长获量下滑 | 中 | 活跃↑ 新增↓ |
