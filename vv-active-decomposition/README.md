# VV渠道活跃度LMDI-I分解模型

## 快速开始

```bash
# 运行模型（使用默认日期：上周同期 vs 昨天）
npm start

# 指定日期范围
node scripts/run-decomposition.js --startDate 2026-06-09 --endDate 2026-06-15

# 指定输出文件
node scripts/run-decomposition.js --outputFile ./results/analysis.json
```

## 参数说明

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--startDate` | 上期日期（YYYY-MM-DD） | 上周同期 |
| `--endDate` | 本期日期（YYYY-MM-DD） | 昨天 |
| `--outputFile` | 输出文件路径 | `vv_active_decomposition_results.json` |
| `--channel` | 渠道名称 | VV |

## 结果解读

### 五因子含义

1. **会话总量 (C)** - 用户总活跃次数
2. **板块结构 (σ)** - 语音区/视频区的会话占比
3. **时段结构 (τ)** - 24小时的会话分布
4. **单会话时长 (ρ)** - 平均每个会话的时长
5. **平台结构 (φ)** - iOS/Android的时长占比

### 贡献解读

- **正值**：该因子推动总时长增长
- **负值**：该因子拖累总时长增长
- **占比>100%**：正向贡献超过总变化
- **占比<-100%**：负向拖累超过总变化

## 使用场景

1. **日报分析** - 每日运行，监控活跃度变化
2. **周报分析** - 周环比对比，识别趋势
3. **活动评估** - 活动前后对比，评估效果
4. **渠道对比** - 多渠道横向对比

## 注意事项

1. 需要CDP Proxy连接到已登录的浏览器
2. 数据来源：产品数据页面的活跃设备Tab
3. 分时数据使用理论分布估算
4. 结果文件会被覆盖，请及时备份
